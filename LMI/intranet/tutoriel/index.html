Tutoriel

Dans ce tutoriel, nous allons voir pas à pas comment modifier le site.

Prérequis

  • Installer R
  • Installer Rstudio
  • Dans Rstudio, installer les packages suivants:
install.packages("rmarkdown")
install.packages("knitr")
install.packages("dplyr")
install.packages("ggplot2")
install.packages("plotly")
install.packages("blogdown")
install.packages("rcrossref")
blogdown::install_hugo()

Ca y est, vous êtes prêts à commencer à éditer le site !

Quelques notions de base

Le site utilise un langage simplifié pour produire des contenus riches, le markdown. Voilà une page qui montre les principales caractéristiques.

Comprendre l’arborescence des fichiers

Ce qui nous intéresse est dans le dossier content:

Chacun des sous-dossiers contient des fichiers *.md (markdown) ou *.Rmd (Rmarkdown) qui correspondent au contenu du site. Les fichiers *.en.(R)md correspondent à la version anglaise du fichier correspondant.

A priori, vous n’avez PAS à toucher à cette arborescence. Lors d’une utilisation quotidienne, vous n’aurez qu’à modifier le tableau des personnels, ajouter un article ou un évènement, etc., mais pas à toucher à la structure du site. Nous verrons par la suite comment faire chacune de ces choses.

Compiler le site pour le voir en local avant upload sur le serveur

Nous avons besoin de Rstudio pour compiler tous ces fichiers *.md en un tas de fichier *.html. Lancez Rstudio et ouvrez le fichier build_site.R qui doit se trouver à la racine du site.

Nous devons d’abord dire à R dans quel dossier nous sommes. Chez moi, cela donne cela :

setwd("/Users/colin/Travail/Site LMI/Site LMI/")

Sur Windows, il se peut que l’arborescence ait une autre structure. Tapez getwd() pour savoir où vous êtes et voir cette structure. Une fois que vous avec tapé la bonne arborescence, sauvez build_site.R et vous n’aurez plus à y toucher.

Pour contruire le site, chargez d’abord la librairie blogdown:

library(blogdown)

Vous pouvez maintenat construire le site et le voir dans le navigateur (qui se lance tout seul) :

serve_site()

ATTENTION : avec serve_site(), seuls les fichiers qui ont été édités seront re-compilés.

Si vous avez modifié le fichier content/annuaire/people.csv, tous les fichiers *.Rmd doivent être recompilés, car ils lisent ce fichier .csv pour afficher leur contenu (annuaire, pages persos, listes de publications). Dans ce cas deux options :

  1. Vous n’avez changé qu’une ou deux lignes dans people.csv (typiquement, ajouté un DOI). Vous pouvez éditer le fichier index.Rmd de la personne correspondante, le modifier et le sauver, ce qui forcera sa recompilation.
  2. Vous avez fait des modifications extensives et voulez tout recompiler, lancer :
build_site(local = FALSE)

Cette seconde option sera (beaucoup) plus longue : pour chaque page perso, on retrouve les métas-données de toutes les publis via la liste de DOI qui est donnée dans people.csv, il faut donc le temps que les requettes se fassent.

Ajouter/Supprimer une personne

Les pages personnelles sont stockées dans content/personnel/nom_prenom/. Pour supprimer une personne, supprimez son dossier.

Pour ajouter une personne, dupliquez un dossier existant et renommez-le nom_prenom, sans espace, sans majuscule, sans accent, et en remplaçant les tirets “-” par des underscores “_”, i.e. Jean-Loïc Du Genøût $\rightarrow$ du_genout_jean_loic.

Vous pouvez mainenant éditer le header (le haut du fichier, entre les balises ---) des fichiers index.Rmd et index.en.Rmd qui ont cette tête là :

---
title : "Jean-Loïc DU GENØÛT"
subtitle : "MCF-UCBL"
tags : ["Personnel", "TMP"]
photo: "du_genout_jean_loic.jpg"
display_photo: true
date: 2019-02-21
---

```{r, include=FALSE}
source(file.path("..","..","annuaire","read_people.R"))
path   <- file.path("..","..","annuaire")
people <- read_people(path, "people.csv")
```

```{r, echo=FALSE}
display_pers_infos(people, "ANDRIEUX", english=FALSE)
```

### Intérêts

- Les multimatériaux
- Mais surtout aux interfaces

### Biographie

- 2000: Thèse
- 1994: Bac S

```{r, echo=FALSE, results="asis"}
print_ref_list(NOM="ANDRIEUX", path=path, file="people.csv")
```

Dans la balise tags, laissez “Personnel” et changez l’équipe correspondante. Si vous avec une photo, placez là dans le dossier de la personne et indiquez le nom du fichier. S’il n’y a pas de photo dans le dossier, un espace blanc sera mis à la place.

Les balises de code sont là pour faire apparaître les infos de la personne et sa liste de publications à partir de la liste de DOI trouvée dans people.csv.

Les personnes qui le souhaitent peuvent décrire leurs intérêts, parcours, etc, en ajoutant des sous-titre (e.g ### Biographie) et leur blabla dessous. La commande print_ref_list imprime la liste de publications, donc mettre ce blabla au dessus, comme dans l’exemple ci-dessus.

Pour que la personne apparaisse dans l’annuaire, vous devez l’ajouter dans le fichier content/annuaire/people.csv. Le plus simple est d’ouvrir le fichier content/annuaire/people.xlsx, de l’éditer et de le sauver, puis de le sauver aussi au format .csv.

Ajouter/Supprimer un énènement, un emploi, une formation ou une publication

Comme pour les personnels, il suffit d’ajouter/supprimer un dossier dans content/events/ (en dupliquer un existant et le renommer) (ou content/emplois/, ou content/formations/, ou content/publications/)

Pour ajouter une image en en-tête, mettre dans le dossier une image appelée featured.jpg ou featured.png.

Ajouter un fichier à télécharger

Cas d’une publication

Mettez le fichier que vous voulez rentre disponible au téléchargement dans le dossier static/files/, puis ajoutez dans le header l’entrée url_pdf = "files/monfichier.pdf".

Cas d’un lien

Mettez le fichier que vous voulez rentre disponible au téléchargement dans le dossier courant. Pour faire un lien : [ce qui s'affiche](monfichier.pdf)

Déployer le site

Ça y est, vous êtes content de ce que vous avez fait, vous voulez maintenant mettre le site en ligne. Pour cela, copiez tout le contenu du dossier public sur le serveur du CNRS (utiliser FileZilla, par exemple).

Et voilà !
